# 2026 美加墨世界杯纪实漫画制作规则 v1

## 作品定位

这是一套基于现实世界杯事件的纪实漫画连载。所有赛事、人物、比分、时间线、晋级结果、红黄牌、VAR、伤病、发布会发言、出征和抵达等事件，都应来自已经发生并可核验的公开资料。

漫画创作负责把现实事件转译成画面节奏、情绪张力和分镜叙事，不虚构比赛事实。

## 发布规格

```text
发布场景：社媒滑图阅读
页面比例：3:4 竖图
页面形式：完整漫画页，接近 PDF 漫画书单页质感
单集长度：1-5 页漫画为主
封面策略：普通事件不默认制作封面；重点章节、重大事件、合集或分卷可制作封面
默认风格：黑白日系足球纪实漫画
```

## 章节系统

```text
序章：世界杯之前
第一卷：小组赛风暴
第二卷：淘汰赛之门
第三卷：八强与四强
终卷：决赛与余音
特别篇：冷门、告别、争议、纪录
```

每个事件都归入一个章节。整部作品采用“全景群像 + 事件主角制”：世界杯整体是群像叙事，每一集根据现实事件确定临时主角。

## 事件工作流

默认分阶段执行。除非用户明确说“快速完成”“一次性输出”或“跳过补充”，否则在输出事实资料包后暂停，等待用户补充关键词或确认继续。

```text
1. 输入事件点
2. 官方/权威来源核验事实，同时执行中文媒体/平台初筛
3. 输出事实资料包和初始中文平台/中文媒体雷达
4. 暂停并询问用户是否补充关键词或素材角度
5. 根据补充关键词进行二次素材搜索
6. 必须扩展中国平台素材雷达：微博、抖音、小红书、B站
7. 建立比赛素材池
8. 筛选适合漫画化的元素
9. 事实分层
10. 故事大纲提炼
11. 漫画页数判断
12. 分镜脚本设计
13. 图片 prompt 输出
14. 事实与漫画化边界说明
```

### 输入事件点

用户提供关键词、主题、阶段或具体事件，例如：

```text
世界杯开赛前，各个国家队开始出征
```

## 双来源资料策略

资料分成两套系统：

```text
官方来源定事实，中文媒体/平台与互联网素材定画面、热度和中文语境。
```

```

```

素材来源只能用于发现素材、判断热度、寻找画面灵感和捕捉情绪，不能单独用来确认关键事实。非权威来源不得覆盖官方事实。

### 中国平台素材雷达

只要任务涉及从网上查询事件或素材资料，就必须执行中国平台素材雷达。第一轮事实核验时就要同步做平台初筛，不要等到用户补充关键词之后才开始查中文平台。微博、抖音、小红书、B站四个平台都要单独查询和记录；不能用泛新闻搜索、英文社媒搜索或官方事实搜索替代。

平台分工：

```text
微博：热搜词、超话、媒体号、足协/国家队官微、记者账号、评论区重复句、争议标签
抖音：短视频爆点、画面钩子、剪辑节奏、评论热梗、球员反应、球迷二创、话题标签
小红书：观赛场景、城市打卡、穿搭/球衣文化、旅行笔记、球迷日记、训练基地/场馆攻略、生活方式化表达
B站：解说标题、复盘视频、战术分析、球迷剪辑、长视频叙事角度、标题梗和高频弹幕语境
```

当无法直接进入 App 内搜索时，至少使用公开网页索引查询，并说明“只能覆盖可索引公开内容”。推荐查询模板：

```text
site:weibo.com <事件/球队/球员> <出征|抵达|机场|训练|热搜|超话>
site:douyin.com/video <事件/球队/球员> <出征|抵达|训练|热梗|世界杯>
site:xiaohongshu.com/explore <事件/球队/球员> <观赛|打卡|机场|穿搭|训练基地>
site:bilibili.com/video <事件/球队/球员> <大名单|出征|抵达|解说|复盘|热梗>
```

如果四个平台的公开索引结果很薄或噪声过高，必须在同一轮补做中文媒体兜底搜索。使用百度式/open-web 中文查询，覆盖央视网、新华社/新华网、人民日报/人民网、新浪、搜狐、腾讯、网易、澎湃、懂球帝、虎扑等中文媒体或社区。兜底搜索只能补充中文语境和素材线索，不能替代微博、抖音、小红书、B站四个平台的尝试。

输出时必须把中国平台素材雷达单独列出，字段包括：

```text
平台
查询词或来源
主体
场景/钩子
事实置信度：已确认 / 待核验 / 仅社媒情绪
漫画价值
```

事实边界：

```text
中国平台素材主要用于发现画面、热度、语气和观众情绪。
具体事实如名单、伤病、出发时间、抵达地点、采访原话、比分、红黄牌、晋级结果，必须回到官方、权威媒体或多源交叉验证。
未核验内容可以进入“社媒情绪”或“漫画表达”层，但不能写成确认事实。
```

### 用户补充检查点

完成事实资料包后，先向用户展示确认事实，并询问是否有补充关键词、素材角度或想重点搜索的画面。默认必须在这里暂停，等待用户回答后再继续素材搜索、素材池、故事大纲、分镜和 prompt。

可询问：

```text
事实资料包先确认到这里。你有没有想补充的关键词、人物、镜头、热梗、采访、争议点或社媒讨论方向？我可以据此做二次素材搜索。
```

用户可能补充：

```text
某球员赛后哭了
教练拥抱
替补席反应
看台球迷
某个热搜词
解说名句
赛前通道镜头
赛后发布会
某张照片/视频截图
```

如果用户明确要求快速完成，或任务只是要一次性输出脚本，可跳过这个检查点。

## 比赛素材池

在事实资料包之后、故事大纲之前，建立素材池。素材池不是必选清单，只是候选材料。

```text
核心比赛事件：进球、点球、扑救、红牌、VAR、绝杀、伤停补时
场地环境元素：球场、城市、球员通道、更衣室、草皮、灯光、奖杯
人物反应元素：教练席、替补席、看台、门将、队友、对手、家属席、记者席
比赛机制元素：VAR 屏幕、换人牌、补时牌、点球点、中圈、边裁旗、裁判哨子
赛事叙事元素：卫冕、首冠、告别、复仇、淘汰路径、历史纪录
可用特写：眼神、汗水、球鞋、手套、队长袖标、球网、横梁、比分牌
```

### 筛选原则

根据事件气质筛选素材，只加入能增强叙事、节奏或事实感的元素。不为了丰富而添加无关画面。

每个候选元素问三件事：

```text
1. 这个元素是否来自真实比赛/现场资料？
2. 它是否帮助读者理解事件？
3. 它是否增强这一页的情绪或节奏？
```

如果答案不明显，就不放。

### 网络资料收集

根据事件点检索公开资料。优先使用 FIFA、赛事官方、国家队/足协官方、权威体育媒体、赛后发布会、球员公开社媒、现场图片或视频报道。

每个事件至少记录 2-3 个事实来源，方便后期回查。

### 事实分层

```text
确认事实：谁、何时、在哪里、发生了什么
背景信息：为什么重要，与世界杯进程有什么关系
漫画可用情绪：出征、告别、压力、荣耀、未知、爆发、退场等
```

### 故事大纲

把资料整理成可画的事件结构，而不是新闻流水账。大纲应提炼开场、推进、转折、爆点和余韵。

### 页数判断

```text
小事件：1 页
中事件：2-3 页
重大事件：4-5 页
超级事件：拆成上下集
```

单集不宜过长，避免社媒滑图阅读压力过高。

## 视觉圣经

### 页面规格

```text
比例：3:4 竖图
形式：完整漫画页
阅读：从上到下，优先左到右；如使用日漫式动线，需要用构图明确引导
每页格数：3-7 格
主格：每页至少 1 个大主格
```

### 分镜语言

- 避免平均九宫格、普通四格或整齐六格。
- 常用结构为大远景主格、窄条反应格、动作格、表情特写。
- 重大瞬间可以使用半页主格或跨页感主格。
- 每页至少包含两种镜头节奏：远景、近景、动作、静默。
- 禁止所有分格都横向堆叠。每页尽量包含至少一个竖向特写格或斜切动作格。

### 黑白质感

- 高对比黑白。
- 粗墨线外轮廓。
- 网点阴影。
- 速度线。
- 草皮线、汗水、尘土、灯光和看台声浪用漫画符号表现。
- 不使用彩色，不使用照片质感。

### 镜头库

```text
机场出征：登机口、行李箱、队服箱、球员背影、窗外航线
比赛现场：俯视跑位、脚下球、门将视角、球网、横梁、替补席
情绪特写：眼神、汗水、握拳、低头、背影、空座位
群像场景：看台、城市灯光、国旗海、记者席、球迷酒吧
```

### 角色画法

- 真实人物漫画化。
- 不追求照片级复刻，但要追求真实足球识别度。
- 保留身份线索：真实队名、球员姓名、位置、发型轮廓、体态、已确认球衣号码、球衣颜色与经典图案转译、旗帜色彩、庆祝动作和比赛角色。
- 脚本中可以写真实姓名；图片 prompt 中可以写真实队名和球员名，同时用“漫画化轮廓 + 发型/体态 + 球衣颜色/图案 + 位置/动作”增强识别，而不是要求照片级复刻。

### 场景画法

- 球场要有空间感。
- 观众席可用密集黑白点阵表现。
- 城市、机场、更衣室、发布会、训练场都可以作为世界杯事件场景。
- 非商业纪录/同人漫画可以画真实球队视觉识别、球衣颜色、经典图案、旗帜、球员漫画化特征；避免赞助商 logo、水印和官方赛事 logo，除非用户提供授权素材或明确要求。

### 文字系统

```text
语言：中文
文字量：每页主要文字 5-8 个信息点以内
标题：可使用竖排大字制造冲击
对白：短句为主
旁白：少量、纪实、克制
拟声：用于射门、奔跑、哨声、看台声浪
```

文字应该像漫画画面的一部分，而不是新闻说明书。

### 身份锚点

少放文字，但必须保留让读者进入现实事件的身份锚点。

```text
必须优先保留：球队名、关键人物名、关键分钟、比分/结果
可以保留：短旁白、短对白、拟声字
尽量减少：长解释、完整新闻句子、战术说明
```

每个文字块尽量短。优先使用标题牌、比分牌碎片、球衣背名、旁白框、人物名标签。涉及真实球员时，脚本里可以写真实姓名；图片 prompt 中可写人物名，并用漫画化轮廓、发型、体态、位置和动作来表达识别度，但不要要求照片级复刻。

生图模型容易生成变形文字。正式发布时，建议让图像生成负责画面、分镜、留白和少量身份锚点，再由后期人工覆盖准确文字。

可以使用已确认且对识别有帮助的球衣号码。避免让模型随机生成未确认号码；如果号码不重要或不确定，再在 prompt 中加入：

```text
No random or unverified jersey numbers.
Leave clean caption boxes for team names, player name, minute, and score.
```

## 分镜版式基准

默认不要使用全横向堆叠。用于日系足球漫画页的基准版式：

```text
上半页：一个大横向主格，占页面约 40-45%，展示球场空间、跑位、压迫或事件环境。
中段：一条窄横向反应格，占页面约 10-15%，展示队友、对手、教练、球迷或记者反应。
下半页左侧：一个大斜切动作格，占页面约 30-35%，展示突破、射门、扑救、登机、奔跑等关键动作。
下半页右侧：两个上下堆叠的竖向特写格，展示眼神、脚下球、球网、门柱、手套、护照、行李牌、比分牌等细节。
```

Prompt 中可明确写：

```text
Do not stack all panels as horizontal strips.
Do not make a regular grid.
Use varied panel shapes: one wide top panel, one thin reaction strip, one large diagonal action panel, two stacked vertical close-up panels.
```

### 禁止事项

- 不要彩色。
- 允许真实球队视觉识别：队名、球衣颜色、经典图案、旗帜、球员漫画化特征。
- 避免赞助商 logo。
- 避免水印。
- 避免官方赛事 logo，除非用户提供授权素材或明确要求。
- 不要长段解释文字。
- 不要把一页做成单张插画。
- 不要平均格子排版。
- 不要虚构比分、进球、红牌、VAR、晋级结果或赛后影响。

## Prompt 基础风格片段

```text
black-and-white Japanese football documentary manga, complete 3:4 vertical manga page, non-uniform panel layout, one large main panel with smaller reaction and close-up panels, high contrast ink linework, screentone shadows, speed lines, expressive panel borders, clear reading order, cinematic football storytelling, Chinese short captions and speech bubbles only, no color, real team names, recognizable national-team kit colors and patterns translated into manga tones, flags, player manga likeness through silhouette, hairstyle, posture, role, and iconic football body language, avoid sponsor logos, watermarks, and official tournament logo marks
```

## Prompt 文字规则片段

```text
Chinese text only, large and legible. Include only these exact text items: "<team>", "<opponent>", "<player>", "<minute>", "<score>". No other text. Leave clean caption boxes if exact text placement is important. Use confirmed jersey numbers only when they help recognition; no random or unverified jersey numbers.
```
